# 🧱 Modularização

## 📝 Descrição do Projeto
Este projeto apresenta uma proposta de atividade teórico-prática voltada ao ensino de Pensamento Computacional e Modularização. O objetivo central é aplicar a filosofia de "Dividir para Conquistar", resolvendo problemas complexos através da decomposição em partes menores e mais gerenciáveis.
Desenvolvida para o Departamento de Ciência da Computação pelo Prof. Diego Marques de Carvalho, a atividade desafia duplas de estudantes a projetarem a arquitetura de um software de forma unplugged (sem o uso de computadores), priorizando a lógica estruturada e o design de software sobre a sintaxe de programação. O cenário prático escolhido é um Sistema de Caixa e Troco, onde se deve calcular a menor quantidade de notas para devolver ao cliente.

## 🚀 Tecnologias e Ferramentas Utilizadas

Metodologia: Pensamento Computacional (Decomposição, Reconhecimento de Padrões, Abstração e Algoritmos).
Ferramentas Físicas: Papel, caneta e lógica pura.
Técnicas de Design: Top-Down Design, Fluxogramas e Pseudocódigo estruturado.
Conceitos de Software: Princípio DRY (Don't Repeat Yourself), Alta Coesão e Baixo Acoplamento.

## 📊 Resultados e Aprendizados
A atividade foca no desenvolvimento de competências de arquitetura distribuída e "Clean Code".
Abstração de "Caixa Preta": Aprendizado sobre como criar funções autônomas onde o usuário não precisa conhecer o funcionamento interno para utilizá-las.
Divisão de Responsabilidades: Separação clara entre o Integrador (responsável pelo fluxo geral e interface) e o Especialista (responsável pela lógica interna dos módulos).
Interface e Contrato: Definição rigorosa de parâmetros de entrada e tipos de retorno antes da implementação isolada.
Eficiência Lógica: Implementação de módulos sugeridos como validar_pagamento(), calcular_troco() e decompor_notas().

## 🔧 Como Executar
Por ser um desafio unplugged, a execução não depende de compiladores, mas sim de um processo colaborativo:
Forme uma dupla: Divida os papéis entre Integrador e Especialista.
Defina o Contrato: Acordem os parâmetros e retornos das funções necessárias para o sistema de troco.
Desenvolva no Papel: Crie o fluxograma detalhado e o pseudocódigo estruturado.
Valide o Design: Verifique se o código evita repetições (DRY) e se cada função realiza apenas uma tarefa (Coesão).
